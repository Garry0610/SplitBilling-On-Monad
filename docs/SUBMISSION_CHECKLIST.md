# Submission Checklist — BuildAnything "Spark" Hackathon

Copy these into the submission form when ready.

- [ ] **Name:** SplitBill
- [ ] **Description:** 1–2 sentences, plain language, no jargon
- [ ] **Problem:** copy from README section 2
- [ ] **Solution:** copy from README section 3
- [ ] **Project URL:** hosted frontend (Vercel/Netlify etc.)
- [ ] **Github repo:** public, README complete, real commit history starting
      after hackathon kickoff
- [ ] **Category:** Monad Testnet or Mainnet — if deployed on both, list Mainnet
- [ ] **Contract address:** deployed SplitBill address
- [ ] **Demo video:** ≤ 3 min, publicly viewable link
- [ ] **Post URL:** public social post about the project (needed for "Most
      viral solution")

## Anti-red-flag pass (do this before submitting)

- [ ] No hardcoded/mock data anywhere a judge might click
- [ ] "Pay" button triggers a real transaction, not a toast
- [ ] Paid/unpaid status reads from the contract, not local state
- [ ] UI fits in the viewport, doesn't look like a default AI template
- [ ] Repo has a real README, not just a title
- [ ] Commit history starts after hackathon start time, no giant single
      "final final v2" commit
- [ ] Confirmed eligibility: 18+, not in an excluded region
